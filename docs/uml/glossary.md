# Глоссарий: «Клеточный футбол»

Предметная область мобильного приложения по [SRS.md](SRS.md). Норматив игровой логики — [game-rules.md](game-rules.md) (v1.0), если файл есть в репозитории; иначе ориентир — краткое резюме в SRS §5.

| Термин | Определение | SRS / правила |
|--------|-------------|----------------|
| **Партия (матч)** | Одна завершённая или текущая игра между двумя сторонами на одном поле до наступления условий окончания по правилам v1.0 (голы, ничья, невозможность продолжения и т.д.). В приложении у партии есть счёт, состояние поля и очередь хода. | §3.1, §5, §7.5; game-rules |
| **Поле** | Прямоугольная клетчатая сетка, на которой ведётся игра: узлы (клетки), проведённые линии ходов, положение мяча, ворота на короткой стороне. В SRS v1.0 зафиксирован ориентир **20×28** клеток, ворота шириной **6**. | §4.1, §5; game-rules |
| **Клетка** | Узел сетки поля с целочисленными координатами; точка, из которой строятся отрезки хода и в которой может находиться мяч. | §4.1, §5; game-rules |
| **Мяч** | Текущая позиция игры на поле (обычно привязана к клетке или узлу сетки в модели); перемещается по мере применения допустимых ходов. | §7.1 FR-APP-03; game-rules |
| **Ход** | Законченное действие стороны: построение ломаной линии по правилам (в т.ч. обычный ход из **трёх** клеток по резюме SRS §5) с проверкой ограничений на пересечения и касания линий. Ход завершается после корректного применения системой. | §5, §7.5 FR-ENG-01, FR-ENG-02; game-rules |
| **Сегмент пути (линия хода)** | Элементарный отрезок между соседними клетками в составе хода; на поле отображается как часть нарисованной траектории. Обычный ход в модели может включать несколько сегментов согласно правилам. | §5; game-rules |
| **Игрок (участник партии)** | Сторона, совершающая ходы: человек за устройством или роль «второй игрок» при локальном PvP. В модели может храниться имя/метка стороны и признак, человек это или алгоритм. | §2.3, §7.3, §9.1; game-rules |
| **Искусственный интеллект (ИИ)** | Встроенный алгоритм соперника: автоматически выбирает **допустимый** ход по тем же правилам, что и человек. Не является отдельным человеком-актором, но участвует в партии как вторая сторона. | §1.3, §7.2 FR-AI-02 |
| **Режим игры** | Способ организации партии: **против ИИ**, **локальный PvP на одном устройстве** или сценарий, связанный с **турниром** (ведение сетки и результатов). Онлайн-PvP отнесён к Phase 2 и в v1.0 не реализуется. | §3.1–3.2, §7.2–7.4 |
| **Очередь хода** | Правило, определяющее, какая сторона сейчас должна выполнить ход; после успешного хода очередь передаётся сопернику (с учётом особых фаз по правилам). Интерфейс обязан явно показывать, чей ход (особенно в PvP). | §7.3 FR-PVP-02; game-rules |
| **Гол** | Событие забитого мяча в ворота соперника; ведёт к обновлению счёта и процедурам ввода мяча по правилам (например, с линии центра после гола — по game-rules). | §5; game-rules |
| **Штанга** | Попадание/касание конструкции ворот, приводящее к отскоку или особой процедуре по правилам v1.0; должно быть отражено в логике партии. | §5; game-rules |
| **Штрафной удар** | Отдельный тип хода/фазы (в т.ч. длина **7** клеток в резюме SRS §5), задаваемый нормативом game-rules; применяется при нарушениях или специальных ситуациях согласно документу правил. | §5; game-rules |
| **Фаза «7 ходов»** | Специальная фаза партии после исчерпания точек продолжения в описанных условиях; ограничивает дальнейшее развитие партии до **семи** ходов или иных исходов по правилам. | §5; game-rules |
| **Ничья** | Исход партии без победителя в описанных правилами случаях (в т.ч. ничья **0–0** при выполнении условий из SRS §5). | §5; game-rules |
| **Турнир** | Режим учёта соревнования между несколькими участниками на клиенте v1.0: список участников, выбранный формат сетки, набор матчей турнира и их статусы. Не требует сети. | §7.4 FR-TRN-01, FR-TRN-02 |
| **Участник турнира** | Человек или команда в списке турнира, представленная **отображаемым именем** (никнейм); не путать с «игроком партии» внутри одной игры на поле. | §7.4 FR-TRN-01 |
| **Матч турнира** | Пара участников турнира и запись о результате (победитель, счёт — в объёме, согласованном с правилами турнира в приложении). Может не запускать автоматически партию на поле, если в реализации турнир только журнал результатов. | §7.4 FR-TRN-03 |
| **Формат турнира** | Способ построения сетки: **олимпийка (круговая система)** или **плей-офф на выбывание**; задаётся при создании турнира и влияет на список матчей. | §7.4 FR-TRN-02 |
| **Недопустимый ход** | Попытка завершить ход, нарушающую ограничения разделов правил (например, пересечение/касание линий — по game-rules). Система **не применяет** такой ход и показывает понятное сообщение пользователю. | §7.3 FR-PVP-03, §7.5 FR-ENG-02; game-rules |
| **Сохранение сессии** | Локальное сохранение на устройстве данных **незавершённой партии** и **турнира** при штатном закрытии приложения (NFR-03); пользователь может начать новую партию без восстановления предыдущей через явное действие в UI. | §8 NFR-03 |
| **Главное меню** | Стартовый экран приложения с переходами к игре, правилам, турниру и т.д.; в репозитории — мокап `mockups/01-glavnaya.png`. | §6.1, Приложение A SRS |
| **Экран выбора режима** | Экран выбора **ИИ / локальный PvP / турнир** (и связанных действий); мокап `mockups/02-rezhim.png`. | §6.1, Приложение A SRS |
| **Вариант использования (Use Case)** | Описываемый в UML сценарий взаимодействия актёра с системой (например, «Играть против ИИ»); в ЛР 3–4 для каждого UC оформляется **поток событий** в `use-case-scenarios.md`. | Методичка ЛР 3–4, §9 SRS |
